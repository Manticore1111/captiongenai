REGISTRATION FLOW:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

User Form Input:
  ├─ Full Name: "John Doe"
  ├─ Email: "john@example.com"
  ├─ Password: "SecurePass123!" (8+ chars, uppercase, lowercase, numbers, symbols)
  ├─ Phone: "+31612345678" (optional)
  └─ Agree Terms: ✓

                        ↓

Frontend Validation:
  1. Check name filled
  2. Check email format
  3. Check passwords match
  4. Check password length ≥ 8
  5. Check password strength

                        ↓

Send to Backend:
  POST http://localhost:5000/api/auth/register
  {
    "name": "John Doe",
    "email": "john@example.com",
    "password": "SecurePass123!",
    "phone": "+31612345678"
  }

                        ↓

Backend Processing:
  1. Validate all fields
  2. Check email uniqueness (database constraint)
  3. Check password strength
  4. Hash password with bcryptjs (10 salt rounds)
  5. Store in SQLite database
  6. Generate JWT token
  7. Return user data + token

                        ↓

Database Storage:
  users table:
  ├─ id: 1
  ├─ email: "john@example.com" (UNIQUE)
  ├─ password: "$2a$10$..." (hashed & salted)
  ├─ name: "John Doe"
  ├─ phone: "+31612345678"
  ├─ country: null
  ├─ subscription_status: "free"
  ├─ created_at: "2026-04-13 14:30:00"
  └─ ...

                        ↓

Frontend Session:
  localStorage.setItem('vcai_auth_token', token)
  localStorage.setItem('vcai_user_data', JSON.stringify(user))

                        ↓

Redirect: → dashboard.html ✓


LOGIN FLOW:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

User Form Input:
  ├─ Email: "john@example.com"
  └─ Password: "SecurePass123!"

                        ↓

Frontend Validation:
  1. Check email filled
  2. Check password filled

                        ↓

Send to Backend:
  POST http://localhost:5000/api/auth/login
  {
    "email": "john@example.com",
    "password": "SecurePass123!"
  }

                        ↓

Backend Processing:
  1. Query database: SELECT * FROM users WHERE email = "john@example.com"
  
  IF user NOT found:
    ❌ Return: "Invalid email or password. User not registered."
    Status: 401
  
  IF user found:
    2. Compare passwords with bcrypt.compare()
    
    IF password DOESN'T match:
      ❌ Return: "Invalid email or password."
      Status: 401
    
    IF password MATCHES:
      3. Generate JWT token
      4. Update last_login & streak
      5. ✅ Return: user data + token
      Status: 200

                        ↓

Frontend Session:
  IF success:
    localStorage.setItem('vcai_auth_token', token)
    localStorage.setItem('vcai_user_data', JSON.stringify(user))
    Redirect: → dashboard.html ✓
  
  ELSE:
    Show error message ❌
    Stay on login.html


WHAT'S PREVENTED:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Fake/Invalid Logins:
  ❌ Email: fake@example.com → "User not registered"
  ❌ Email: john@example.com + Password: wrong123 → "Invalid password"
  ❌ Multiple registrations with same email → UNIQUE constraint error

Plain Text Passwords:
  ❌ Password is never stored as plain text
  ✅ Password hashed: bcryptjs (salted 10 rounds)
  ✅ Safe comparison: bcrypt.compare()

Unauthorized Access:
  ❌ Can't access dashboard without valid token
  ❌ Token checked on every protected page load
  ✅ Invalid token → redirect to login.html

Session Hijacking:
  ✅ Tokens are JWT (cryptographically signed)
  ✅ Tokens contain user ID & email
  ✅ Token expires (can add expiration)
  ✅ localStorage is protected by browser same-origin policy
